项目名称：词汇训练器


版本名称：a0.9.1
开发周期：alpha

功能描述：
VocabularyTrainer.py
随机单词抽取：程序会从用户指定的JSON文件中随机抽取一个单词，并显示其含义。
用户输入与检查：用户需要输入他们认为的单词，程序会检查用户的输入是否与随机抽取的单词匹配。
学习进度记录：如果用户答对，程序会在一个新的JSON文件（learned_words.json）中记录该单词，表示用户已掌握。如果用户没有答对，程序就会提示错误
文件路径设置：用户可以通过界面按钮选择并设置词汇数据的JSON文件路径。
图形用户界面：提供了一个简洁的图形用户界面，方便用户进行操作。
VocabularyRecorder.py
按照VocabularyTrainer.py所需要的json文件格式储存单词

相对于a0.9：
修复了一些隐患
优化了部分代码
新增了VocabularyTrainer.py

注意事项与隐患：
学习进度覆盖：每次用户答对单词时，程序都会覆盖learned_words.json文件。这意味着之前的学习记录会被新的记录替换，而不是累加。如果需要记录所有已掌握的单词，请考虑修改此部分逻辑。
界面尺寸固定：主窗口的尺寸是固定的（750x390），在某些分辨率的屏幕上可能不是最佳显示效果。您可以根据需要自行调整界面尺寸。
异常处理不足：当前版本的程序在异常处理方面可能不够完善。例如，当指定的文件不存在或无法读取时，程序可能会崩溃或显示不友好的错误信息。建议增加更全面的异常处理逻辑，以提升用户体验。

使用方法：
确保有一个包含词汇数据的JSON文件，并按照格式要求组织数据。
运行此脚本，并通过“File path options”按钮设置词汇数据的文件路径。
使用“Fetch Random Word”按钮获取随机单词，并尝试猜测其对应的词汇。
使用“Check Answer”按钮检查您的答案。

注意事项与隐患：
文件路径问题：请确保在设置文件路径时选择了正确的JSON文件。错误的文件路径可能会导致程序无法正确读取词汇数据。
数据格式问题：请确保您的词汇数据JSON文件按照预期的格式进行组织。不符合预期格式的数据可能会导致程序出错。
学习进度覆盖：每次用户答对单词时，程序都会覆盖learned_words.json文件。这意味着之前的学习记录会被新的记录替换，而不是累加。如果需要记录所有已掌握的单词，请考虑修改此部分逻辑。

异常处理不足：当前版本的程序在异常处理方面可能不够完善。例如，当指定的文件不存在或无法读取时，程序可能会崩溃或显示不友好的错误信息。建议增加更全面的异常处理逻辑，以提升用户体验。
